Why FastAuth

Everything auth. Zero boilerplate.

Stop rewriting auth logic on every project. FastAuth handles the complete authentication stack — as a pure JSON API — so you focus on your product.

Plug & Play

One include_router call registers 14 auth endpoints. Works with any frontend framework.

Pure JSON API

Every endpoint returns JSON. No HTML pages, no redirects. Your frontend owns the UI.

JWT + Refresh Tokens

Per-token jti blacklisting, automatic rotation on refresh, configurable expiry.

bcrypt & argon2

Best-in-class password hashing. Timing-safe verify. Switch hasher with one config value.

Social OAuth2

Google, GitHub, Discord. Auto-create users from OAuth profiles. One config dict.

RBAC

Built-in role and permission system. Gate any route with one dependency function.

Email Verification

Auto-send on register. Dev mode shows token in email for instant testing. Production: point at your SPA.

Password Reset

Token-based, 1-hour expiry. Dev mode email shows raw token + curl command. Anti-enumeration by design.

Rate Limiting

Sliding-window brute-force protection on login. Configurable attempts and lockout window.

Multi-ORM

Tortoise ORM, SQLAlchemy async and sync. Auto-detected at runtime from your model class.

Cookie / Body / Both

Refresh tokens in body, httpOnly cookie, or both. One config value.

CLI Scaffolding

fastauth --default-setup postgresql --async generates a complete project in seconds.

Installation

Choose your ORM

FastAuth auto-detects Tortoise ORM and SQLAlchemy from your model class. Pass get_db for SQLAlchemy; omit it for Tortoise.

Base only

pip install fastauth-framework

Tortoise ORM recommended async

pip install "fastauth-framework[tortoise]"

SQLAlchemy 2.x

pip install "fastauth-framework[sqlalchemy]"

SQLModel

pip install "fastauth-framework[sqlmodel]"

All extras

pip install "fastauth-framework[all]"

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:

RBAC — protect routes with roles

from fastapi import Depends
from fastauth.exceptions import PermissionDeniedError

def require_role(*roles):
    async def dep(user = Depends(auth.get_current_user())):
        for role in roles:
            if role not in (user.roles or []):
                raise PermissionDeniedError(f"Role '{role}' required")
        return user
    return dep

@app.get("/admin")
async def admin_panel(user = Depends(require_role("admin"))):
    return {"message": f"Hello admin {user.username}"}

@app.get("/mod")
async def mod_panel(user = Depends(require_role("admin", "moderator"))):
    return {"message": "Mod panel"}
API Reference

All 14 endpoints — pure JSON

Mounted under /auth by default (configurable via router_prefix). Every endpoint returns JSON. No HTML, no redirects.

MethodEndpointAuthDescription
POST/auth/register—Create account; sends welcome or verification email if configured
POST/auth/login—Login with username or email + password; returns token pair
GET/auth/meBearerReturn current authenticated user
POST/auth/refreshRefresh tokenIssue new token pair; old refresh token immediately revoked
POST/auth/logoutBearerBlacklist current access token, clear cookies
POST/auth/change-passwordBearerChange password — old password required
POST/auth/reset-password—Request reset email; always 200 (anti-enumeration)
POST/auth/reset-password/confirm—Confirm with {"token":"…","new_password":"…"}
GET/auth/verify-email?token=…—Verify email inline — returns JSON confirmation
POST/auth/verify-email—Verify email with token in body {"token":"…"}
POST/auth/resend-verificationBearer or {"email":"…"}Resend verification email; no auth needed if email provided in body
POST/auth/revoke-allBearerRevoke all refresh tokens for this user
GET/auth/{provider}/login—Redirect to OAuth2 provider (google / github / discord)
GET/auth/{provider}/callback—Handle OAuth2 callback; create or login user; return token pair
Email Setup

Password reset & email verification

Without email config the endpoints still work — no email is sent. Tokens can be tested directly via Swagger or curl. Add the email dict when ready for real delivery.

URL Configuration — the right way

FastAuth uses two URL parameters to control where email links point. Both are optional with sensible dev-mode defaults.

verify_email_url
dev
{base_url}/auth/verify-email?token=…
Link goes to backend GET endpoint → returns JSON
prod
https://myapp.com/verify?token=…
Link goes to your frontend page → frontend calls POST /auth/verify-email
reset_password_url
dev
not set → dev email
Email shows raw token + curl example for Swagger testing
prod
https://myapp.com/reset?token=…
Link goes to your frontend page → frontend calls POST /auth/reset-password/confirm

// ── Email verification ───────────────────────────────────────────────────────
// User lands on your SPA verify page with ?token=TOKEN in the URL
const token = new URLSearchParams(window.location.search).get("token");

const res = await fetch("https://api.myapp.com/auth/verify-email", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ token }),
});
// { "message": "Email verified successfully" }

// ── Password reset ───────────────────────────────────────────────────────────
// User lands on your SPA reset page with ?token=TOKEN in the URL
const token = new URLSearchParams(window.location.search).get("token");

// User enters new password, then:
const res = await fetch("https://api.myapp.com/auth/reset-password/confirm", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ token, new_password: "NewPass123!" }),
});
// { "message": "Password reset successfully" }

import httpx

Security

Secure by default

Every security decision is made for you using industry-standard defaults. No configuration required.

bcrypt / argon2

Adaptive cost hashing with timing-safe verify. Switch hashers with one config value.

JWT jti blacklist

Every token has a unique ID. Logout blacklists it instantly. Pluggable store (Redis).

Refresh rotation

Old refresh token is revoked the moment a new pair is issued — replay attacks blocked.

Brute-force protection

Sliding-window rate limiter: 5 failed attempts triggers a 5-minute IP lockout.

Email enumeration guard

Password reset and resend always return 200 regardless of whether the email exists.

Timing attack mitigation

Dummy hash verify runs on unknown usernames to prevent response-time leaks.

Secure cookie flags

httpOnly, secure, sameSite configurable. Safe defaults included.

Short-lived tokens

Access: 15 min. Reset links: 1 hour. Verify links: 24 hours. All configurable.
