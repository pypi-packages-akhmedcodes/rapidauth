Why FastAuth

Everything auth. Zero boilerplate.

Stop rewriting auth logic on every project. FastAuth handles the full authentication stack so you can focus on your product.

Plug & Play

One include_router call wires up register, login, logout, refresh, reset-password and more — automatically.

JWT + Refresh Tokens

Configurable access/refresh expiry, per-token jti blacklisting, automatic rotation on refresh.

bcrypt & argon2

Best-in-class password hashing. Timing-safe verify. Automatic rehash detection when cost factor changes.

Social OAuth2

Google, GitHub, Discord. Auto-create users from OAuth profiles. Just pass your client_id & client_secret.

RBAC — Roles & Permissions

Built-in role and permission system. Gate any route with require_role("admin") in one line.

Email Verification

Auto-send verification emails on register. Resend endpoint included. SMTP config in one dict.

Password Reset

Full flow: request → token → confirm. Tokens expire in 1 hour. Anti-enumeration by design.

Rate Limiting

Sliding-window brute-force protection on login. Configurable max attempts and lockout window.

Multi-ORM Support

Auto-detects Tortoise ORM and SQLAlchemy 2.x. Custom field mapping for any model schema.

Cookie / Body / Both

Deliver refresh tokens in the response body, httpOnly cookie, or both. One config line.

Custom User Models

Bring your own model. Map any field names with username_field, email_field, etc.

CLI Scaffolding

fastauth --default-setup sqlite|sqlalchemy|postgresql generates database.py + models.py instantly.

4
Protect routes with roles 
from fastapi import Depends
from fastauth.exceptions import PermissionDeniedError

def require_role(*roles):
    async def dep(user = Depends(auth.get_current_user())):
        for role in roles:
            if role not in (user.roles or []):
                raise PermissionDeniedError(f"Role '{role}' required")
        return user
    return dep

@app.get("/admin")
async def admin_panel(user = Depends(require_role("admin"))):
    return {"message": f"Hello, admin {user.username}"}

@app.get("/dashboard")
async def dashboard(user = Depends(auth.get_current_user())):
    return {"welcome": user.username}

API Reference

Auto-registered endpoints

All endpoints are mounted under /auth (configurable via router_prefix).

Method Endpoint Auth required Description
POST /auth/register — Create new account, returns token pair
POST /auth/login — Login with username or email + password
GET /auth/me Bearer Return current authenticated user
POST /auth/refresh Refresh token Issue new access+refresh pair, rotate old token
POST /auth/logout Bearer Blacklist current access token
POST /auth/change-password Bearer Change password (requires old password)
POST /auth/reset-password — Request password reset email
POST /auth/reset-password/confirm — Confirm reset with token + new password
POST /auth/verify-email — Verify email address with token
POST /auth/resend-verification Bearer Re-send verification email
POST /auth/revoke-all Bearer Revoke all refresh tokens for this user
GET /auth/{provider}/login — Redirect to OAuth2 provider (google/github/discord)
GET /auth/{provider}/callback — Handle OAuth2 callback, return token pair
Security

Secure by default

Every security decision is made for you — using industry-standard defaults.

bcrypt / argon2

Adaptive cost hashing with timing-safe checkpw. Automatic rehash on upgrade.

JWT jti blacklist

Every token has a unique ID. Logout blacklists it immediately. Pluggable store (Redis, DB).

Refresh token rotation

Old refresh token is revoked the moment a new one is issued. Stolen tokens expire fast.

Brute-force protection

Sliding-window rate limiter: 5 failed attempts triggers a 5-minute IP lockout.

Email enumeration guard

Password reset always returns HTTP 200 regardless of whether the email exists.

Timing attack mitigation

Dummy hash verify runs on unknown usernames to prevent response-time leaks.

Secure cookie flags

httpOnly, secure, sameSite configurable. Defaults to safest values for production.

Token expiry

Access tokens: 15 min. Refresh tokens: 7 days. Both fully configurable.

Email Setup

Password reset & email verification

Without email config the endpoints still work — but no email is sent. Add the email dict and base_url to enable real delivery.

Minimal config 
auth = FastAuth(
    user_model=User,
    jwt_secret="your-secret",

    email={
        "host":       "smtp.gmail.com",
        "port":       587,
        "username":   "you@gmail.com",
        "password":   "xxxx xxxx xxxx xxxx",  # Gmail App Password (16 chars)
        "from_email": "you@gmail.com",
        # optional:
        "use_tls":    True,   # default True
        "use_ssl":    False,  # default False (use True only for port 465)
    },

    base_url="https://myapp.com",         # used to build reset / verify links
    email_verification_required=False,    # True → block login until email verified
)

Password reset flow
POST
/auth/reset-password — send email address
→ FastAuth creates token (1 hour expiry), sends email
  user clicks link in email
POST
/auth/reset-password/confirm
{"token": "...", "new_password": "..."}
→ password updated, token invalidated
Always returns 200 even if email not found (anti-enumeration)
Email verification flow
POST
/auth/register
→ account created, verification email sent
  user clicks link in email (24h expiry)
POST
/auth/verify-email — {"token": "..."}
→ email verified, login unblocked
POST
/auth/resend-verification 🔒 Bearer
→ re-send verification email
Set email_verification_required=True to block login until verified
Email link format 
# Password reset link (sent in email):
{base_url}/auth/reset-password/confirm?token={token}
# Example:
https://myapp.com/auth/reset-password/confirm?token=abc123...

# Email verification link (sent in email):
{base_url}/auth/verify-email?token={token}
# Example:
https://myapp.com/auth/verify-email?token=xyz789...

Your frontend reads ?token= from the URL and calls the confirm endpoint.

CLI

fastauth command line

Scaffold a complete project in seconds. Every setup generates database.py + models.py + main.py. Default mode is --async.

SQLite
fastauth --default-setup sqlite Tortoise ORM + aiosqlite (async, default)
fastauth --default-setup sqlite --async Tortoise ORM + aiosqlite
fastauth --default-setup sqlite --sync SQLAlchemy sync + sqlite3
SQLAlchemy
fastauth --default-setup sqlalchemy SQLAlchemy async + aiosqlite (async, default)
fastauth --default-setup sqlalchemy --async SQLAlchemy async + aiosqlite
fastauth --default-setup sqlalchemy --sync SQLAlchemy sync + sqlite3
PostgreSQL
fastauth --default-setup postgresql SQLAlchemy async + asyncpg (async, default)
fastauth --default-setup postgresql --async SQLAlchemy async + asyncpg
fastauth --default-setup postgresql --sync SQLAlchemy sync + psycopg2
fastauth init Scaffold .env.example and auth_config.py
fastauth version Print installed fastauth-framework version

--sync mode generates synchronous DB code. Sync calls block the FastAPI event loop — use --async for production.

Configuration

FastAuth() parameters

All parameters are optional except user_model and jwt_secret.

Parameter Type Default Description
user_model type required Your ORM User model class
jwt_secret str required Secret key for signing JWTs (min 16 chars)
jwt dict None Full JWT config: secret, algorithm, access_expire, refresh_expire
username_field str "username" Field name for username in your model
email_field str "email" Field name for email in your model
password_field str "password" Field name for hashed password in your model
password_hasher str "bcrypt" "bcrypt" or "argon2"
refresh_token_mode str "body" "body" | "cookie" | "both"
cookie dict {} httponly, secure, samesite, domain
email dict None SMTP config: host, port, username, password, from_email
social_auth dict None OAuth providers: google, github, discord
email_verification_required bool False Block login until email is verified
get_db callable None FastAPI dependency yielding an async DB session (SQLAlchemy)
router_prefix str "/auth" URL prefix for all auth endpoints
base_url str "http://localhost:8000" Base URL used in email verification / reset links
